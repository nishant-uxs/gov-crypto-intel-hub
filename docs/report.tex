\section{Executive Summary}
% ═══════════════════════════════════════════════════════════════

The \textbf{Government Crypto Intelligence Hub} is a comprehensive web-based dashboard for monitoring, analyzing, and reporting on cryptocurrency-related regulatory developments, scam intelligence, and policy frameworks — with a primary focus on India.

\subsection*{Key Metrics}
\begin{itemize}[nosep]
    \item \textbf{29 pages} — 8 public-facing tabs + 15 admin panel pages + API routes
    \item \textbf{13 API endpoints} — RESTful, all database-driven
    \item \textbf{16 database tables} — Fully seeded with sourced intelligence data
    \item \textbf{Zero placeholder data} — Every displayed value originates from the database
    \item \textbf{Zero build errors} — Clean compilation with Next.js 14.2.35
\end{itemize}

\subsection{SQLite Design Decision}

\begin{infobox}[SQLite for Development]
The project uses \textbf{SQLite} as the development database for zero-configuration setup. The database file (\texttt{prisma/dev.db}) is created automatically by \texttt{npx prisma db push}.

\textbf{Migration to PostgreSQL:} Change two lines in \texttt{prisma/schema.prisma}:
\begin{verbatim}
datasource db {
  provider = "postgresql"    // was: "sqlite"
  url      = env("DATABASE_URL")  // postgresql://user:pass@host:5432/db
}
\end{verbatim}
Then run \texttt{npx prisma db push}. All queries are identical — Prisma abstracts the dialect.
\end{infobox}

\subsection{JSON Array Workaround}

SQLite does not support array columns. Fields that would be \texttt{String[]} in PostgreSQL are stored as JSON strings:

\begin{center}
\begin{tabular}{lll}
\toprule
\textbf{Table} & \textbf{Field} & \textbf{Storage Format} \\
\midrule
Exchange & assets, notices, alerts & \texttt{"["BTC","ETH"]"} \\
ScamType & vectors, redFlags & \texttt{"["Fake App","Phishing"]"} \\
AiBrief & sourceItemIds & \texttt{"["id1","id2"]"} \\
\bottomrule
\end{tabular}
\end{center}

API routes parse these back to arrays before responding:
\begin{lstlisting}[language=TypeScript]
const parsed = exchanges.map((ex) => ({
  ...ex,
  assets: JSON.parse(ex.assets || "[]"),
  alerts: JSON.parse(ex.alerts || "[]"),
}));
\end{lstlisting}

\section{Data Flow Architecture}
% ═══════════════════════════════════════════════════════════════

\subsection{Complete Request Lifecycle}

\begin{enumerate}[nosep]
    \item \textbf{Browser} opens \texttt{http://localhost:3000}
    \item \textbf{page.tsx} renders server-side: \texttt{Header} + \texttt{KpiStrip} + \texttt{DashboardTabs}
    \item \textbf{KpiStrip} (server component) calls \texttt{prisma.kpi.findMany()} directly — no API call needed
    \item \textbf{DashboardTabs} mounts on client, renders default tab (\texttt{IndiaIntel})
    \item \textbf{IndiaIntel} calls \texttt{fetch("/api/news?region=INDIA\&page=1")}
    \item \textbf{/api/news} route handler queries \texttt{prisma.newsItem.findMany()} with filters
    \item \textbf{Prisma} translates to SQL, executes against \texttt{prisma/dev.db}
    \item \textbf{JSON response} returns to browser, React \texttt{setState} triggers re-render
    \item User clicks another tab → new component mounts → new API call → new data
\end{enumerate}

\subsection{Two Data Paths}

\begin{tcolorbox}[title=\textbf{Path A: Static Reference Data},colback=blueaccent!5!white,colframe=blueaccent!50!black]
\textbf{Source:} \texttt{prisma/seed.ts}\\
\textbf{Trigger:} \texttt{npx tsx prisma/seed.ts} (one-time)\\
\textbf{Tables:} User, NewsSource, Kpi, Exchange, ScamType, Country, all Advisory* tables\\
\textbf{Data origin:} Real government reports (CBDT, CBIC, FIU-IND, FATF, ED, CERT-In)
\end{tcolorbox}

\vspace{0.5em}

\begin{tcolorbox}[title=\textbf{Path B: Live News Data},colback=greenok!5!white,colframe=greenok!50!black]
\textbf{Source:} \texttt{src/lib/ingest.ts}\\
\textbf{Trigger:} \texttt{npx tsx src/lib/ingest.ts} or \texttt{POST /api/ingest}\\
\textbf{Process:}
\begin{enumerate}[nosep]
    \item Fetch RSS/Atom feeds from 8 configured sources
    \item Parse each feed item (title, summary, body, URL, pubDate)
    \item Check for duplicates by URL + sourceId
    \item Create NewsItem records with region from source config
    \item Auto-tag using Claude Haiku 4 (classifies into POLICY/ENFORCEMENT/COMPLIANCE/SCAM/MARKET/INNOVATION)
    \item Update source's \texttt{lastFetchedAt} timestamp
\end{enumerate}
\end{tcolorbox}

\subsection{Example: News API Implementation}

\begin{lstlisting}[language=TypeScript,caption={src/app/api/news/route.ts}]
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/prisma";

export async function GET(req: NextRequest) {
  const { searchParams } = new URL(req.url);
  const page = parseInt(searchParams.get("page") || "1");
  const region = searchParams.get("region") || "INDIA";
  const search = searchParams.get("search") || "";
  const tag = searchParams.get("tag") || "";
  const pageSize = 20;

  const where: any = { region, isSuppressed: false };
  if (tag) where.tag = tag;
  if (search) {
    where.OR = [
      { title: { contains: search, mode: "insensitive" } },
      { summary: { contains: search, mode: "insensitive" } },
    ];
  }

  const [items, total] = await Promise.all([
    prisma.newsItem.findMany({
      where,
      include: { source: { select: { name: true } } },
      orderBy: [
        { isPinned: "desc" },
        { isBreaking: "desc" },
        { publishedAt: "desc" }
      ],
      skip: (page - 1) * pageSize,
      take: pageSize,
    }),
    prisma.newsItem.count({ where }),
  ]);

  return NextResponse.json({
    items, total,
    totalPages: Math.ceil(total / pageSize),
    page,
  });
}
\end{lstlisting}

\subsection{NextAuth.js Configuration}

\begin{lstlisting}[language=TypeScript,caption={src/lib/auth.ts}]
export const authOptions: NextAuthOptions = {
  providers: [
    CredentialsProvider({
      name: "credentials",
      credentials: {
        email: { label: "Email", type: "email" },
        password: { label: "Password", type: "password" },
      },
      async authorize(credentials) {
        const user = await prisma.user.findUnique({
          where: { email: credentials.email },
        });
        if (!user || !user.isActive) throw new Error("Invalid credentials");
        const isValid = await bcrypt.compare(credentials.password, user.password);
        if (!isValid) throw new Error("Invalid credentials");
        await prisma.user.update({
          where: { id: user.id },
          data: { lastLoginAt: new Date() },
        });
        return { id: user.id, email: user.email, name: user.name, role: user.role };
      },
    }),
  ],
  session: { strategy: "jwt", maxAge: 8 * 60 * 60 },  // 8 hours
  callbacks: {
    jwt: ({ token, user }) => { token.role = user.role; return token; },
    session: ({ session, token }) => { session.user.role = token.role; return session; },
  },
  pages: { signIn: "/admin/login" },
  secret: process.env.NEXTAUTH_SECRET,
};

\section{News Ingestion Pipeline}
% ═══════════════════════════════════════════════════════════════

\subsection{Pipeline Architecture}

\begin{verbatim}
Trigger: npx tsx src/lib/ingest.ts  OR  POST /api/ingest
    │
    ├─► Step 1: Load active sources from DB
    │       prisma.newsSource.findMany({ where: { isActive: true } })
    │
    ├─► Step 2: For each source, fetch RSS feed
    │       rss-parser.parseURL(source.url)
    │
    ├─► Step 3: For each RSS item:
    │       ├─ Check duplicate: prisma.newsItem.findFirst({ url, sourceId })
    │       ├─ Create: prisma.newsItem.create({ title, summary, body, url, ... })
    │       └─ Count ingested items
    │
    ├─► Step 4: Update source timestamp
    │       prisma.newsSource.update({ lastFetchedAt, lastError })
    │
    └─► Step 5: Auto-tag untagged items (if ANTHROPIC_API_KEY set)
            ├─ Fetch 50 untagged items
            ├─ For each: POST to Claude Haiku API

\subsection{Recent Development Updates (2026-06-10)}

The codebase received a short iteration to stabilize live news ingestion and the India tab UX. Key changes applied and verified locally:

\begin{itemize}[nosep]
  \item \textbf{Reliable feeds:} Replaced several failing sources and moved active feeds to BBC, NYTimes, Al Jazeera, CoinDesk, and Cointelegraph. The updater script is \texttt{scripts/update-news-sources.ts} and the seed changes are in \texttt{prisma/seed.ts}.
  \item \textbf{Ingestion hardening:} Added safe date parsing to avoid crashes on malformed pubDate values (see \texttt{src/lib/ingest.ts} — `safePublishedAt`).
  \item \textbf{Auto-fetch on open:} The news API now triggers a conditional ingest when sources are stale (configurable via `AUTO_FETCH_NEWS`) — implemented in \texttt{src/app/api/news/route.ts} (`triggerIngestIfNeeded`).
  \item \textbf{India tab fallback:} If `region=INDIA` returns no items, the API falls back to `GLOBAL` headlines so the India tab always shows recent news (added fallback in \texttt{src/app/api/news/route.ts}).
  \item \textbf{Deactivate failing feeds:} Known failing or blocked feeds (Reuters, CNBC, TheBlock, some AP endpoints) were marked inactive to avoid repeated errors; the updater script deactivates these automatically.
  \item \textbf{Admin login fix:} Resolved an admin redirect loop by introducing a top-level login route and adjusting middleware and NextAuth `pages.signIn` to point to the correct path (see \texttt{src/app/admin/*} and \texttt{src/lib/auth.ts}).
  \item \textbf{Verification run:} After updating sources the ingestion run produced the following sample counts locally: BBC World 41, BBC Business 55, BBC Technology 21, NYTimes Business 49, NYTimes World 56, NYTimes Technology 32, Al Jazeera 25, Cointelegraph 30. Global API and India-fallback were verified via `curl`/`node` requests against \texttt{/api/news}.
  \item \textbf{Next steps available:} Optionally add Indian domestic feeds (The Hindu, Indian Express, Economic Times, NDTV) to the active list and re-run ingestion to populate India-specific items; or enable scheduled ingestion (cron) for continuous updates.
\end{itemize}
